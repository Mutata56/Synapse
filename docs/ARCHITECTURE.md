# Архитектура Synapse

## Стек

- **Интерфейс:** React 18 + TypeScript, сборка на Vite. Стили - Tailwind v4 с трёхслойными дизайн-токенами (примитивы → семантические переменные → утилиты), анимации на framer-motion.
- **Состояние:** Zustand (стора заметок, календаря, тостов и т.д.), без Redux.
- **Редактор:** BlockNote (поверх ProseMirror) с кастомными блоками - выноски, таблицы, галереи, встроенные базы данных, мультиколонки, интерактивная доска.
- **Граф связей:** `graphology` как структура данных + собственный физический движок + Pixi.js (WebGL) для рендера.
- **Интерактивная доска:** собственный движок на Canvas 2D, без сторонних библиотек.
- **Десктоп-оболочка:** Tauri 2 (Rust). Плагины fs / dialog / opener / global-shortcut, HTTP на `reqwest`.
- **Хранение:** обычные Markdown-файлы на диске. Ни базы данных, ни облака.

## Хранение данных

Источник правды - папка `notes/` на диске. Никакой БД (ни SQLite, ни IndexedDB), `localStorage` используется только под мелкие настройки интерфейса (язык, ширина сайдбара, зум редактора), но не под контент.

- **Формат заметки:** один `.md` = frontmatter (`---...---`) + тело Markdown. Во frontmatter - `id`, `title`, даты, иконка, обложка, избранное, настроение, плюс lossless-JSON документа BlockNote и хэш тела (чтобы доверять JSON, только если тело не правили снаружи).
- **id заметки - это путь файла**, а не поле в frontmatter. Теги (`#tag`) и `[[вики-ссылки]]` нигде не хранятся - они выводятся из тела при чтении (код-блоки игнорируются).
- **Структура воркспейса** - служебное прячется в дотфайлы: `.trash/` (корзина), `.assets/` (бинарники картинок), `.versions/` (история), `.templates/`, `.settings.json`, `.folder-meta.json` (цвета папок), `.tasks.json`, плюс mtime-кэши меты. Обход дерева их пропускает.
- **Настройки и цвета папок едут вместе с папкой** - они лежат дотфайлами внутри `notes/`. Скопировал папку - переехали и заметки, и акцентный цвет, и хоткей, и задачи.
- **Дерево** строится рекурсивным обходом с семафором на 48 параллельных чтений и mtime/size-кэшем (не перечитываем неизменившиеся файлы).
- **Картинки:** дедупликация по содержимому - имя файла в `.assets/` это SHA-256 байтов, поэтому два одинаковых импорта дают один файл. В UI отдаются через asset-протокол Tauri (`convertFileSrc`), на перезагрузке URL пересобирается из имени ассета.
- **Корзина** - это перемещение файлов (rename) в `.trash/` с сохранением структуры, а не удаление; восстановление - обратный rename. Коллизии имён разводятся суффиксами, чтобы ничего не перезатёрлось.
- **История версий** - снапшоты-файлы `.versions/<id>/<timestamp>.md`: перед перезаписью с изменённым телом старое содержимое копируется в снапшот. Троттлинг 1 снимок / 10 мин, лимит 50 последних. Откат сначала снимает снапшот текущего состояния (сам обратим).
- **Бэкап/восстановление** - самодельный zip-сборщик (без сторонних библиотек) пакует весь воркспейс. Восстановление кладёт распакованное рядом и делает **атомарный своп**: старый `notes` → `notes.bak-<дата>`, новый на его место; если своп падает - откат. Старая папка не удаляется.

## Граф связей

Глобальный граф связей и локальный граф вокруг открытой заметки.

- **Узлы** - папки и заметки (теги намеренно не узлы). **Рёбра** двух видов: `contains` (структура папка→заметка) и `link` (из `[[вики-ссылок]]`, резолв по заголовку). Граф неориентированный, степень узла задаёт его радиус и массу.
- **Движок раскладки - собственный** (`ForceLayout`), не d3-force и не ForceAtlas2:
  - **Barnes-Hut quadtree** для отталкивания - O(n log n) вместо O(n²).
  - **Verlet-интегратор** (симплектичный, стабильнее явного Эйлера).
  - Силы: кулоновское отталкивание, пружины рёбер (лог-закон Eades), анти-перекрытие, гравитация к центру масс, когезия и разнос кластеров (каждая папка - остров).
  - Раскладка **инкрементальная**: один шаг физики на кадр тикера Pixi, в главном потоке. Симуляция **засыпает**, когда кинетическая энергия падает ниже порога на 40 кадров подряд - в покое процессор не греется.
- **Рендер - Pixi.js (WebGL):**
  - Узлы - векторные диски (чёткие на любом зуме); гало - один раз «запечённая» радиально-градиентная текстура, на узел вешается тонированный спрайт (без покадрового блюра).
  - Все рёбра рисуются одним `Graphics`-объектом.
  - Оболочки кластеров - GPU-metaball: поле плотности рисуется в offscreen-текстуру, кастомный GLSL-фильтр берёт изоповерхность по порогу.
  - Зум/пан через `scale`/`position` контейнера; кинематографичный наезд камеры - экспоненциальный lerp.
- **Оптимизации:** пул ячеек quadtree (ноль аллокаций в установившемся режиме), заморозка симуляции после стабилизации, троттлинг мини-карты и геометрии оболочек, кэш `getBoundingClientRect` (иначе layout-thrash на каждом движении мыши), переиспользование спрайтов (на апдейт - только позиция и тинт, без пересборки геометрии), мемоизация построения графа по дереву.
- **Тема:** мост CSS-переменные → int-тинты Pixi. Цвет парсится через скрытый пробный `<span>` (браузер сам резолвит `oklch`/`var()`/именованные в `rgb`). Смена темы ловится `MutationObserver` и перекрашивает узлы/гало/оболочки **без перестройки графа и сброса раскладки**.

## Интерактивная доска

Собственный движок (~3400 строк vanilla JS); React - тонкая оболочка, всё мутабельное состояние (фигуры, связи, камера, инструмент, выделение, undo/redo) живёт в движке, вне React.

- **Два canvas-слоя:** нижний `world` (сетка + рёбра + спрайты узлов, `pointer-events:none`, двигается GPU-трансформом) и верхний `overlay` (выделение, рамки, marquee, UI связей - ловит все события). World-холст увеличен на буфер 256px с каждой стороны, чтобы мелкий пан был дешёвым CSS-translate без обнажения краёв.
- **Рендер - Canvas 2D** (не WebGL). Один rAF-цикл с dirty-флагом: кадр рисуется только при изменениях, простой ничего не делает.
- **Модель:** 8 типов фигур (перо, прямоугольник, эллипс, стрелка, текст, узлы БД/процесса/заметки) + рёбра, которые хранят концы как ссылки на узлы (а не координаты) - путь сам перестраивается при движении узла. Доска сериализуется в JSON и едет строкой в пропе кастомного блока BlockNote внутри заметки.
- **Оптимизации:**
  - **rAF-gate для pointermove** - события мышей с частотой 1000-8000 Гц коалесцируются в одну обработку за кадр (перо - исключение, ему важна точность штриха).
  - **Кэш спрайтов на фигуру + атлас по визуальному ключу** с подсчётом ссылок и LRU-вытеснением - одинаковые узлы делят один битмап, пан превращается в `drawImage`.
  - **SpatialHash** (ячейка 256 единиц) - рендер и hit-test работают только по видимым/соседним ячейкам, стоимость кадра ~O(видимого), не O(N).
  - **CSS-translate пан с округлением до целых пикселей + `will-change:transform`** - композитинг идёт по GPU-пути.
  - **RDP-упрощение** штриха пера при коммите, кэш сглаженного пути и AABB в WeakMap, O(1)-индексы фигур вместо линейного поиска.
  - **Debounce записи 600 мс**, чтобы автосейв доски не громил историю редактора.
- **Интеграция:** кастомный блок BlockNote, в заметке - компактная карточка с превью и кнопкой «Открыть»; полноэкранный режим - модалка через портал; экспорт доски в PNG теми же painter-функциями (WYSIWYG).

## Rust-бэкенд (минимальный)

Rust намеренно тонкий - делает только то, что нельзя из вебвью. Файлами он **не** занимается.

- **Ровно 5 команд:** `set_capture_shortcut` (переназначить глобальный хоткей на лету), `fetch_ics` (GET `.ics` в обход CORS вебвью), `caldav_discover` / `caldav_put` / `caldav_delete` (PROPFIND/PUT/DELETE в Яндекс.Календарь, свой мелкий XML-парсер без зависимостей).
- **Глобальный хоткей + окно быстрой заметки + трей:** по `Ctrl+Shift+N` поднимается безрамочное окно поверх всех; закрытие главного окна = сворачивание в трей (чтобы хоткей продолжал работать), реальный выход - через меню трея.
- **Вся файловая система - из TypeScript** через `@tauri-apps/plugin-fs` со скоупом `$APPDATA`. Rust файловые операции не проксирует.
- **Плагины:** fs, dialog, opener, global-shortcut; HTTP на `reqwest`.

## Поиск

Постоянного поискового индекса на диске нет. Fuse.js (нечёткий) - только в командной палитре по действиям и заголовкам. Полнотекстовый поиск по телам - `indexOf` с in-memory кэшем, который сам инвалидируется по `updatedAt` (перечитываются только изменённые заметки). Поиск внутри заметки подсвечивает совпадения через CSS Custom Highlight API, не вставляя теги в документ.
